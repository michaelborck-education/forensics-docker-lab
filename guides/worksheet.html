üìö Why E01 Format?

  • Industry Standard: Expert Witness Format used by law enforcement and forensic professionals worldwide
  • Court Admissible: Includes built-in hash verification, metadata, and case information
  • Compression: Reduces storage while maintaining bit-for-bit accuracy
  • Tool Support: Compatible with EnCase, FTK, X-Ways, Autopsy, Sleuth Kit, and more
  • Integrity Checking: Internal checksums verify data hasn't been corrupted or tampered with

CLI Analysis with Sleuth Kit

Sleuth Kit requires mounting IMG files with ewfmount first.

Mount IMG with ewfmount
List partitions with mmls
List files with fls
Identify deleted files (marked with *)
Recover deleted files with tsk_recover
# Enter the forensic workstation docker compose run --rm dfir # [You are now inside the forensic lab environment] # Mount the E01 file (required for Sleuth Kit) mkdir -p /tmp/ewf ewfmount /evidence/cloudcore_suspect_usb.E01 /tmp/ewf # Verify mount worked ls /tmp/ewf # Should show: ewf1 # Check partitions in the image mmls /tmp/ewf/ewf1 # Get file system information fsstat /tmp/ewf/ewf1 # List all files including deleted (marked with *) fls -r -d /tmp/ewf/ewf1 > /cases/file_list.txt # View the file listing cat /cases/file_list.txt | less # Look for deleted files (marked with *) grep "\\*" /cases/file_list.txt # Recover all deleted files mkdir -p /cases/recovered tsk_recover /tmp/ewf/ewf1 /cases/recovered/ # Extract a specific file by inode (replace [INODE] with actual number) icat /tmp/ewf/ewf1 [INODE] > /cases/recovered_file # Search for keywords in recovered files strings /cases/recovered/* | grep -i "password\|confidential\|secret" # Exit the workstation when done exit
Combined Approach (Recommended)

Use both tools to cross-verify findings for court-ready analysis.

Best Practice Workflow:

  1. Start with Autopsy for visual exploration and automatic processing
  2. Note file paths, inode numbers, and suspicious artifacts
  3. Use Sleuth Kit CLI to extract specific files for detailed analysis
  4. Verify hash values match between both tools
  5. Document all findings with screenshots and command outputs
Important: Always verify findings with multiple tools. Courts expect thorough verification and may question single-tool analysis.
